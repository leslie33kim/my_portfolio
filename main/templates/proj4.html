Graphical User Interface(GUI)

A GUI was used to allow the user to interact with the vehicle both autonomously and manually. Features included in the GUI created via LabVIEW as shown in Figure 3 were as follows:

Figure 3. RoboTaxi GUI designed in LabVIEW

  • Pre-programmed track selection
    • Track 1 drives the vehicle on a triangular track
    • Track 2 drives the vehicle on a square track
  • Manual control with emergency stop
    • Drive forward
    • Drive backward
    • Turn left
    • Turn right
  • Two selectable obstacle avoidance modes
    • Track 2 Mode avoids objects while attempting to remain on the track
    • Dynamic Mode avoids objects by turning left or right depending on object location
  • Plot of the LiDAR clustered point clouds
    • Displays the clustered point clouds, with different colors indicating clusters
    • Red dot in each cluster indicates the centroid of the obstacle
    • Uses DBSCAN point clustering algorithm and k-nearest neighbor algorithm to optimize the epsilon value(maximum distnace between points in same cluster) and minimum number of points in clusters
  • Debugging tools, which include the response from the commands sent via TCP/IP, the IP address, and buttons to stop the TCP and LiDAR

Multitasking was implemented by running two while loops in parallel in LabVIEW as shown in Figure 5. For further control, these two while loops could be replaced with two timed loops, where we can further customize the specific processor core the loop is running at, e.g. both loops should run at processor core number zero as shown in Figure 6.
